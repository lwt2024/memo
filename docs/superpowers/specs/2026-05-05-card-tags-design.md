# 卡片标签功能设计文档

## 概述
为记忆卡片系统添加标签功能，支持用户为卡片添加1-3个标签，便于分类和筛选。

## 功能需求

### 1. 数据模型
- **Tag 模型**：存储标签信息
  - id: 唯一标识
  - name: 标签名称
  - color: 标签颜色（十六进制）
  - userId: 所属用户
  - isPreset: 是否为预设标签
  - createdAt: 创建时间
  
- **CardTag 模型**：卡片与标签的多对多关联
  - cardId: 卡片ID
  - tagId: 标签ID
  - userId: 用户ID（用于快速查询）
  
- **Card 模型**：添加 tags 关联

### 2. 标签管理

#### 2.1 预设标签
- 系统提供6-8个预设标签：
  - 重要（红色）
  - 待复习（橙色）
  - 重点（蓝色）
  - 易错（紫色）
  - 已掌握（绿色）
  - 拓展（青色）
  - 基础（黄色）
  - 进阶（粉色）

#### 2.2 自定义标签
- 用户可以创建自己的标签
- 输入标签名称（2-20字符）
- 选择颜色（从预设颜色中选择）
- 自定义标签与用户绑定，其他用户不可见

### 3. 卡片编辑
- 在卡片编辑界面添加标签选择区域
- 显示所有可用标签（预设 + 用户自定义）
- 已选择的标签高亮显示
- 最多选择3个标签
- 支持取消选择
- 添加"新建标签"按钮，弹出创建标签对话框

### 4. 标签显示
- **彩色小标签样式**：圆角矩形背景，白色文字
- 在卡片列表中显示在卡片元数据旁边
- 显示1-3个标签，超出部分不显示或显示"..."

### 5. 标签筛选
- 在筛选栏中显示该卡片组中所有已使用的标签
- 点击标签可以筛选显示包含该标签的卡片
- 支持多选标签（显示同时包含所有选中标签的卡片）
- 支持"全部"选项，清除筛选
- 显示每个标签下有多少张卡片

## 技术实现

### 数据库更新
- Prisma schema 更新
- 数据库迁移

### 后端 API
- POST `/tags` - 创建标签
- GET `/tags` - 获取用户的所有标签
- POST `/cards/:id/tags` - 为卡片添加标签
- DELETE `/cards/:id/tags/:tagId` - 移除卡片的标签
- GET `/decks/:id/tags` - 获取卡片组中使用的标签

### 前端组件
- TagSelector 组件 - 标签选择器
- TagDisplay 组件 - 标签显示组件
- TagFilter 组件 - 标签筛选组件

## UI 设计

### 标签样式
- 圆角矩形，背景色为标签颜色
- 白色文字，字体适中
- 有悬停和选中状态

### 卡片列表布局
```
[序号] 卡片正面内容...
       [标签1] [标签2] [标签3]  📅 日期  🔄 复习次数  [掌握程度]
```

### 筛选栏布局
```
搜索: [输入框]  X
排序: [下拉]  [↑/↓]
掌握程度: [下拉]
标签: [全部] [重要(5)] [待复习(3)] [重点(8)] ...
```

## 实现优先级
1. 数据模型和数据库迁移
2. 后端 API
3. 标签管理和选择 UI
4. 卡片列表标签显示
5. 标签筛选功能
